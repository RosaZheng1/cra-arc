When to use

This pattern supports topics that have a discrete number of task-oriented pages. The persistent menu walks the user through a suite of related tasks.

The general information architecture (IA) path is:

  • [General background]
  • [Considerations]
  • [Main action needed]
    • [After main action]
  • [Contact]

The above flow does not suggest that any one item has to be:

  • Presented on a single page, as specific sections may be so complex that they need to be split over several pages, or even have sub-pages.
  • Repesented in the IA at all, as not all topics require everyone of these tasks.

What to avoid

This pattern is not to be used:

  • When you simply want the persistent navigation feature. Unless there is a strong chance the user will need to use most or all of the menu items, presented in an order to create a progressive understanding, do not use this pattern.
  • When topics that are so large in scope, that the persistent navigation or sub-pages balloon into a massive GC subway site. This pattern excels when offering a discrete number of limited pages that all relate to a single product or concept.

How to implement

  1. Create a main index page

    Use as the landing page for a set of related pages that use the subway navigation.

    • Navigation:
      • Links should be task-focused
      • Links should be presented in an order that allows for progressive understanding
      • If a contact link is offered, it should be the last item
      • Link text needs to comply with Services and information - Canada.ca design pattern: Link text
      • Link descriptions needs to comply with Services and information - Canada.ca design pattern: Link descriptions

  2. Create sub-pages

    Sub-pages can be either a first or second level of IA.

    When a first level page links its second level of IA, you select the design option suited for that content, outlined below. Within the same single product, you may end up different first level page and second level patterns.

    If the page is a content-only page and has no links to a secondary level of IA

    This is the typical and most commonly used pattern.

    • Navigation:
      • In the persistent menu, indicate the current page as active
      • Paginate to the next first level of IA, except on the final page
      • In the breadcrumb, the main index page needs to show as the parent page
    • Content:
      • Use the Table of contents component, if needed
        • page, where there is no pagination
      • Use any other standard design component and function, as appropriate

    If the page is a navigation-only page to a sequential secondary level of IA

    First level of IA

    If a user needs to be stepped through the sub-level IA in a sequencial order, mimic the same linear navigation design found on the main index page.

    • Navigation:
      • Links should be task-focused
      • Links should be presented in an order that allows for progressive understanding
      • In the persistent menu, indicate the current page as active
      • Link text needs to comply with Services and information - Canada.ca design pattern: Link text
      • Link descriptions needs to comply with Services and information - Canada.ca design pattern: Link descriptions
      • In the breadcrumb, the main index page needs to show as the parent page
      • Do not include a pagination button
      • Do not repeat the second level IA links in persistent side menu, as they are already featured in the main content area
    Second level of IA

    Use when a user needs to be stepped through the sub-level IA in a sequencial order.

    • Navigation:
      • In the persistent menu, indicate the current page as active
      • Paginate to the next second level of IA, except on the final sub-page, paginate back up the the next first level IA
      • In the breadcrumb, the first level IA page needs to show as the parent page, preceeded by the main index page
    • Content:
      • Use the Table of contents component, if needed
      • Use any other standard design component and function, as appropriate

    If the page is a navigation-only page to a non-sequential secondary level of IA

    First level of IA

    Use the doormat design if the user would likely only need one of the navigation items or if there is no natural sequence to the navigation items.

    • Navigation:
      • Links should be task-focused
      • In the persistent menu, indicate the current page as active
      • In the breadcrumb, the main index page needs to show as the parent page
      • Do not include a pagination button
      • Do not repeat the second level IA links in persistent side menu, as they are already featured in the main content area
    • Content:
      • Link text needs to comply with Services and information - Canada.ca design pattern: Link text
      • Link descriptions needs to comply with Services and information - Canada.ca design pattern: Link descriptions
    Second level of IA
    • Navigation:
      • In the persistent menu, indicate the current page as active
      • Paginate to the next first level of IA
      • In the breadcrumb, the first level IA page needs to show as the parent page, preceeded by the main index page
    • Content:
      • Use the Table of contents component, if needed
      • Use any other standard design component and function, as appropriate

    If the page has authoritative content and also has links to the secondary level of IA within that content

    First level of IA

    This pattern is rarely used. The design ensures the core content is given immediate profile, while still offering the ability to reference periphal supporting content within the IA. The supporting content on the second level of IA is typically appendix-like and has a limited audience.

    • Navigation:
      • In the persistent menu, indicate the current page as active
      • Display the second level of IA in the GC subway design
      • Paginate to the next first level of IA
      • In the breadcrumb, the main index page needs to show as the parent page
    • Content:
      • Within the content area, link to the second level of IA at the point of need
      • Use the table of contents component, if needed
      • Use any other standard design component and function, as appropriate
    Second level of IA
    • Navigation:
      • In the persistent menu, indicate the current page as active
      • Paginate to the next first level of IA
      • In the breadcrumb, the first level IA page needs to show as the parent page, preceeded by the main index page
    • Content:
      • Use the Table of contents component, if needed
      • Use any other standard design component and function, as appropriate
