Icons – Standard web pages

Use icons to draw attention to specific types of information in a body of text, usually to denote direct and immediate actions to be taken.

How to use

Icons are glyphs − picture-based fonts, similar to wingdings. They do not convey meaning, unlike traditional fonts. Therefore, real text must accompany the icon to ensure accessibility requirements are met.

Correct use

  • Use one of the two icon sets, Font Awesome or Glyphicons for decorative purposes
  • Use icons when assisting people with site functionality, with elements such as:
    • buttons
    • alerts
    • form fields
    • external or email links
    • links to non-HTML files like a PDF or Excel files
  • Accompany all icons with text
    • The text can be invisible if needed, but must be present for accessibility reasons
  • Include a space between the icon and the text or link following it, so they are easy to read

Incorrect use

  • Don't use to add decoration to navigation labels or headings
  • Don't use to add emphasis to text
  • Don't use checkmark () or X mark () icons, except for lists. Refer to: Add icons for checkmarks and X marks
  • Don't use to style links internal to Canada.ca
  • Don't use to represent a third party's logo, as this can be contrary to that company's brand asset policy. For instance:
    • Facebook's do's and don'ts states: "Use any icons, images or trademarks to represent Facebook other than what is found on this resource center."
    • Twitter's brand assets and guidelines states: "Our logo is always either blue or white. The Twitter bird is never shown in black or other colors."
    • Pixel-perfect spacing and other considerations are also required. It is best, legally-speaking, to use the actual third party logo image

Coding considerations:

  • Use icons within a <span> tag
  • Avoid using extra small icons

Stacked icons

Appearance

fa-phone (inverse) on fa-circle (solid)

fa-users on fa-circle (regular)

fa-ban on fa-camera

HTML code 
<!-- fa-phone (inverse) on fa-circle (solid) -->
<span class="fa-stack fa-lg">
  <span class="fa fa-circle fa-stack-2x"></span>
  <span class="fa fa-phone fa-stack-1x fa-inverse"></span>
</span> [ Text ]

<!-- fa-users on fa-circle (regular) -->
<span class="fa-stack fa-lg">
  <span class="far fa-circle fa-stack-2x"></span>
  <span class="fa fa-phone fa-stack-1x"></span>
</span> [ Text ]

<!-- fa-ban on fa-camera -->
<span class="fa-stack fa-lg">
  <span class="fas fa-camera fa-stack-1x"></span>
  <span class="fas fa-ban fa-stack-2x text-danger"></span>
</span> [ Text ]

Coding considerations:

  • To stack multiple icons, use the fa-stack class on the parent, the fa-stack-1x class for the regularly sized icon, and fa-stack-2x for the larger icon.
  • The fa-inverse class can be used as an alternative icon colour. You can also add larger icon classes to the parent to further control the sizing.
