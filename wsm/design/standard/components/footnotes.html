Footnotes – Standard web pages

Use a footnote to note a reference, citation, or comment that provides details about something on that page. It is non-critical content that relates to the main text, and is always at the bottom of the page.

On this page

  • How to implement
  • Complementary components and functions

How to implement

The purpose of the Footnotes plugin is to implement a consistent, accessible way of handling footnotes across Government of Canada web sites. The main concept behind this solution is to place footnotes in a definition list, within a dedicated section. Supporting CSS is used to lay out the footnotes and hide navigational aids.

Correct use

  • Place at the bottom of the page
  • Consistently use the header "Footnotes" even if there is only a single footnote
  • Number all footnotes
  • Use only the number along with the term "footnote"
    • This ensures the link text is meaningful
    • For example, use "2 footnote" instead of just "2"
  • Add the word "footnote" in invisible text so screen readers know the link is a footnote

Incorrect use

  • Do not use any other terms such as "Endnote", as this can confuse users
  • Do not place multiple "referrer" return links beside a footnote
    • The code knows to take the user back to the last footnote source a user selected
    • For example, if a user selects the second instance of Footnote 1, the referrer link takes them back to this point in the content
