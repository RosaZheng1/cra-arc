Tabbed interface – Standard web pages

Use to separate related content into different panes that are viewable one at a time when the respective tab is clicked.

On this page

  • When to use
  • How to implement
  • Configuration options and events
  • Complementary components and functions

When to use

Use the tabbed interface pattern to group and present different aspects of content that are closely related, such as:

  • Geographic places, where the tabs can be used to present content relating to a place’s location, demographics, economy, etc.
  • Product catalogues, where the tabs can be used to present consistent facets of each product in the catalogue, such as specifications, features, etc.

Module tabs [a User Interface (UI) design pattern where content is separated into different panes, and each pane is viewable one at a time] are meant to contain modular and easily digested information, and as such, module tabs are only appropriate for data that is succinct and to the point such as lists, data conveyed as graphics (data visualization), or one to two short paragraphs of text.

Smashing Magazine
Module Tabs in Web Design: Best Practices and Solutions (external link)

Correct use

  • Use tabs to show alternative views (modules) of the same types of information
  • Ensure that tab headers are clear and concise; avoid lengthy labels
  • When repeating the tabbed interface across a set of related pages (for example, on country pages), always ensure the tab headers are presented consistently
  • Make the default active tab first on page load; but another tab may be used as the active tab on page load if required
  • Ensure tab labels appear as a single row, and do not wordwrap over multiple lines, regardless of the viewport/resolution
  • Ensure tab labels display/hide content on the same page, and do not navigate to different pages
  • Label tabs in a parallel naming structure for example:
    • Tax forms
    • Benefit forms
    • Charity forms

Incorrect use

  • Don't use to hide primary or essential content
  • Don't use to tidy up the look of a page
    • If the page is too busy, separate the content over several pages, or place it in smaller blocks of information
  • Don't use to take a user through a process that moves from tab to tab
    • This can result in the need to check back and forth between tabs
    • Tabs are meant to experience in isolation of each another
  • Don't use when content must be presented in a specific sequential order (instead, use a steps list)
  • Don't use when content needs to be compared and contrasted, which would lead to repeated switching between tabs (instead, use tables or other components that allow all content to be presented at once)
  • Don't use when content is lengthy or detailed, because of space limitations in the tabbed interface
  • Don't use when content presentation includes only 2 tabs (for greater efficiency, use subheadings)
  • Don't use labels that are not parallel in naming structure for example:
    • Forms
    • Regional offices
    • Apply for benefits

Appearance

Content of first tab.

Link to open a specific tab: show the third tab.

HTML code 
<div class="wb-tabs">
	<div class="tabpanels">
		<details id="details-panel1">
			<summary>Example 1</summary>
			<p>Content of first tab.</p>
			...
		</details>
		<details id="details-panel2">
			<summary>Example 2</summary>
			<p>Content of second tab.</p>
			...
		</details>
				<details id="details-panel3">
			<summary>Example 3</summary>
			<p>Content of third tab.</p>
			...
		</details>
	</div>
</div>

<!-- Link to open a specific tab -->
<p><a href="#details-panel3" class="wb-tabs-ext">Show the third tab</a></p>
