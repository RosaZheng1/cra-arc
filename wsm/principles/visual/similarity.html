Law of similarity

On this page

  • Overview
  • Examples
  • Take aways

Overview

Definition: We group similar things together. This similarity can occur in the form of shape, colour, shading or other qualities.

In practice: Users notice matching forms and shapes, colors and lines. Design components can work to enhance the content, and can also confuse it. Users perceive items with the same characteristics (similar elements) as related, even if this is not the intention.

Examples

Correct use

  • In the following example, the faceted search on the right side is given its own unique colour and design. Every element that is part of the search is treated visually the same.

Incorrect use

  • The use of similarity in the nex example is incorrect, as different lines of content have the same background colour applied. This gives the impression the content is related, when it is not. This could confuse users.
  • In this next example, all different filed inputs look the same regardless of the length and format of information requested. This could confuse users. The date field only requires YYYY-MM-DD, but the field width is the same as for every other free-format answer.
  • The content in the following example looks like a single infographic, when it's actually three tabs, each with circles that take the user to different content pages.
