Monitor and refine

After the web pages launch, the Data and Usability Analysis group will continue to monitor the following data:

  • feedback
  • surveys
  • call drivers
  • analytics
  • search results

The goal is to determine:

  • whether implemented changes are having the intended effect
  • if there any new issues to be addressed

Project data, such as task success rates, is tracked using the Usability Performance Dashboard (UPD). Task success rates should be tracked and reported quarterly, biannually, or annually.

Identifying post-launch issues

Depending on whether post-launch issues are deemed to be minor or major, they will be addressed in a specific way:

  • minor updates required to IA (for example, navigation) or other content (for example, key definitions missing), can be implemented and sent for approval to subject matter experts or project teams, depending on the project
  • major issues (for example, reworking the flow of content for findability) will require the return to the planning or design phases for prototyping
