Start with the most important information: CRA web service manual

Start your content with the most important information, and be direct. When people scan web pages, they tend to start in the top left hand corner and scan to the right and down. As they move down the page, they scan less and less to the right.

Inverted pyramid concept

Organize your content using an inverted pyramid structure:

Model

  1. Start with need to know content

    Start with the most important idea, step or information

  2. Continue with top task and core content

    This includes:

    • add details and tasks in order of importance
    • keep all task related content in the centre of the page
    • use links to direct people to key information and tasks

  3. Provide supportive content

    Include information for audiences who want to learn more or have more knowledge of the topic or service, but only if it makes sense

  4. End with peripheral content (usually not needed)

    Add links only if they relate to content that supports completion of the task on the web page (for example, laws or publications)

There are several benefits to writing content this way.

Using the inverted pyramid style can:

  • Improve comprehension: Users can quickly form a mental model and a general understanding of the article, making it easier to understand the details that follow.
  • Decrease interaction cost: Users can understand the main point of the page without having to spend a lot of time reading.
  • Encourage scrolling: This structure can encourage scrolling by engaging the audience with the main point, and drawing them in to the details that follow.
  • Structure content logically: Starting with broad information sets the stage for what follows. Elements like anchor or jump links can become unnecessary when content is structured to draw the user down the page.
  • Support readers who skim: Readers can stop reading at any point on the page and still come away with the main point.

Importance of a good page heading

When a user is scanning, the first piece of content is the page heading. A good heading is important because it sets the expection for what is on the page.

The page heading also becomes the title of the page, which is what search engines use in search results. This is important because poor headings produce poor search results.

Prioritizing information on a navigation page

Navigation pages should:

  • Order items in a logical sequence, if there is a flow or natural order to the navigation links
  • Prioritize the most popular navigation links first
  • Do a mix of the above, if needed

Prioritizing information on a content page

The core content must always follow the inverted pyramid concept.

On longer content pages, there are additional components you can add to expose important content, without forcing the user to scroll and explore the page:

  • Add a lead-in page description
  • Add a table of contents

Adding a lead-in page description

A lead-in page description provides a snapshot of what the page is about. It is placed directly below the page's heading.

Although it is optional on content pages, there is value in having one on longer content pages:

  • It provides the user a sense of what the page is about. This can help them decide if they should invest time into reviewing the rest of the page.
  • Google will display a short description below every link on its search result page. Google does not get this from the metadata and instead will pull a sentence from the content on your page. When there is a lot of content, it may pull a less-than-desirable statement and use that to describe your page in search results. While there is no guarentee that Google will select your lead-in text, if often does.

Adding a table of contents

When a page has multiple content blocks that are grouped under various sub-headings, a table of contents helps the user jump down to a specfic section. The table of contents helps them determine, with little to no scrolling needed, whether that page might have the content they are seeking.

Consider page length

You have to find the right balance between creating pages that are too long or too short.

  • When a page is too long, it can:
    • Overwhelm the user
    • Create undesirable search engine results, due to the lack of focused content
  • When a page is too short, it can:
    • Leave the user thinking they have the full picture, when they do not
    • Leave the user thinking they do not have the full picture, yet they may be uncertain how to get to the related content

There is no hard-and-fast rule to determine the perfect page size. However, you can use effective content structure to help group and organize content. This will help you determine how to break up content, which will support you in determining what to put on a page, and when to create a suite of pages.

What do users hate more: long pages full of information that require lengthy scrolling or having to drill down through multiple—albeit shorter—pages to find what they need? Ask them, and they’ll often say, "We hate scrolling." But usability tests say otherwise. Folks don’t seem to mind scrolling, as long as they believe that it will eventually pay off.

In the trade-off between hiding content below the fold or spreading it across several pages, users have greater success when the content is on a single page. However, if you’re going to go with longer pages, you’d better make sure that your users can easily find the information they need.

Create pages that are easy to scan. That have a consistent structure. That allow users to jump right to where they want to be.
