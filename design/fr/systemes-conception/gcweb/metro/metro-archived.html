Quand l'utiliser

Ce modèle convient aux sujets qui comportent un certain nombre de pages axées sur les tâches. Le menu permanent guide l'utilisateur dans une série de tâches connexes.

Le chemin général de l'architecture de l'information (AI) est le suivant :

  • [Contexte général]
  • [Points à considérer]
  • [Principale mesure à prendre]
  • [Après la mesure principale]
  • [Contact]

Le parcours ci-dessus ne laisse pas supposer que chacun de ces éléments doit :

  • être présenté sur une seule page, car certaines sections peuvent être si complexes qu'elles doivent être réparties sur plusieurs pages ou même sur des sous-pages
  • absolument faire partie de l’AI, car certains sujets n’ont pas besoin de comporter toutes ces tâches

    • Quoi éviter

    Ce modèle ne doit pas être utilisé dans les situations suivantes :

    • Lorsque vous souhaitez simplement utiliser la fonctionnalité de navigation permanente. N’utilisez pas ce modèle, sauf s’il y a de fortes chances que l'utilisateur ait besoin d'utiliser la plupart ou la totalité des éléments du menu, présentés dans un ordre permettant une compréhension progressive.
    • Lorsque les sujets sont tellement vastes que la navigation permanente ou les sous-pages se transforment en un énorme site de style métro. Ce modèle offre d’excellent résultats lorsqu'on propose un nombre limité de pages qui se rapportent toutes à un seul produit ou concept.

    Comment l’appliquer

    1. Créer une page d’index principale

      À utiliser comme page d'accueil pour un ensemble de pages connexes qui utilisent la navigation de style métro.

      • Navigation :
        • Les liens doivent être axés sur les tâches.
        • Les liens doivent être présentés dans un ordre qui permet une compréhension progressive.
        • Si un lien de contact est proposé, il doit être le dernier élément.
        • Le texte des liens doit être conforme aux Services et renseignements - Configuration de conception de Canada.ca.
        • La description des liens doit être conforme aux Services et renseignements - Configuration de conception de Canada.ca.

    2. Créer des sous-pages

      Les sous-pages peuvent correspondre à un premier ou à un deuxième niveau d'AI.

      Lorsqu'une page de premier niveau établit un lien avec son deuxième niveau d'AI, vous sélectionnez l'option de conception adaptée à ce contenu, comme décrit ci-dessous. Dans un même produit, vous pouvez vous retrouver avec des modèles de page de premier niveau et de deuxième niveau différents.

      Si la page est seulement une page de contenu et qu'elle n'a pas de liens vers un deuxième niveau d'AI

      Il s'agit du modèle habituel et le plus souvent utilisé.

      • Navigation :
        • Dans le menu permanent, indiquez que la page actuelle est active.
        • Faites une pagination vers le prochain premier niveau d'AI, sauf pour la dernière page.
        • Dans le fil d'Ariane, la page d'index principale doit apparaître comme la page parent.
      • Contenu :
        • Utilisez la composante Table des matières, si nécessaire.
        • Utilisez toute autre composante et fonctionnalité de conception standard, si nécessaire.

      Si la page est seulement une page de navigation vers un deuxième niveau séquentiel d'AI

      Premier niveau d’AI

      Si l'utilisateur doit parcourir les sous-niveaux de l'AI dans un ordre séquentiel, reproduisez la même conception de navigation linéaire que celle de la page d'index principale.

      • Navigation :
        • Les liens doivent être axés sur les tâches.
        • Les liens doivent être présentés dans un ordre qui permet une compréhension progressive.
        • Dans le menu permanent, indiquez que la page actuelle est active.
        • Le texte des liens doit être conforme aux Services et renseignements - Configuration de conception de Canada.ca.
        • La description des liens doit être conforme aux Services et renseignements - Configuration de conception de Canada.ca.
        • Dans le fil d'Ariane, la page d'index principale doit apparaître comme la page parent.
        • N’incluez pas de bouton de pagination.
        • Ne répétez pas les liens de deuxième niveau de l'AI dans le menu latéral permanent, car ils apparaissent déjà dans la zone de contenu principale.
      Deuxième niveau d’AI

      À utiliser lorsque l'utilisateur doit progresser dans les sous-niveaux de l'AI dans un ordre séquentiel.

      • Navigation :
        • Dans le menu permanent, indiquez que la page actuelle est active.
        • Faites une pagination vers le prochain deuxième niveau d'AI, sauf pour la dernière sous-page, où la pagination revient au prochain premier niveau d'AI.
        • Dans le fil d'Ariane, la page de premier niveau d'AI doit apparaître comme la page parent, précédée de la page d'index principale.
      • Contenu :
        • Utilisez la composante Table des matières, si nécessaire.
        • Utilisez toute autre composante et fonctionnalité de conception standard, si nécessaire.

      Si la page est seulement une page de navigation menant à un deuxième niveau d'AI non séquentiel

      Premier niveau d’AI

      Utilisez la conception par sujets si l'utilisateur est susceptible d'avoir besoin d'un seul des éléments de navigation ou s'il n'y a pas de séquence naturelle entre les éléments de navigation.

      • Navigation :
        • Les liens doivent être axés sur les tâches.
        • Dans le menu permanent, indiquez que la page actuelle est active.
        • Dans le fil d'Ariane, la page d'index principale doit apparaître comme la page parent.
        • N’incluez pas de bouton de pagination.
        • Ne répétez pas les liens de deuxième niveau de l'AI dans le menu latéral permanent, car ils apparaissent déjà dans la zone de contenu principale.
      • Contenu :
        • Le texte des liens doit être conforme aux Services et renseignements - Configuration de conception de Canada.ca.
        • La description des liens doit être conforme aux Services et renseignements - Configuration de conception de Canada.ca.
      Deuxième niveau d’AI
      • Navigation :
        • Dans le menu permanent, indiquez que la page actuelle est active.
        • Faites une pagination vers le prochain premier niveau d'AI.
        • Dans le fil d'Ariane, la page de premier niveau d'AI doit apparaître comme la page parent, précédée de la page d'index principale.
      • Contenu :
        • Utilisez la composante Table des matières, si nécessaire.
        • Utilisez toute autre composante et fonctionnalité de conception standard, si nécessaire.

      Si la page a du contenu qui fait autorité et que ce contenu contient aussi des liens vers le deuxième niveau d'AI

      Premier niveau d’AI

      Ce modèle est rarement utilisé. La conception permet de s'assurer que le contenu principal est directement mis en évidence, tout en offrant la possibilité de faire référence au contenu de soutien périphérique dans l'AI. Le contenu de soutien au deuxième niveau d'AI ressemble généralement à une annexe et a un public limité.

      • Navigation :
        • Dans le menu permanent, indiquez que la page actuelle est active.
        • Affichez le deuxième niveau d'IA dans la navigation de style métro.
        • Faites une pagination vers le prochain premier niveau d'AI.
        • Dans le fil d'Ariane, la page d'index principale doit apparaître comme la page parent.
      • Contenu :
        • Dans la section du contenu, incluez un lien avec le deuxième niveau d'AI aux endroits où cela est nécessaire.
        • Utilisez la composante Table des matières, si nécessaire.
        • Utilisez toute autre composante et fonctionnalité de conception standard, si nécessaire.
      Deuxième niveau d’AI
      • Navigation :
        • Dans le menu permanent, indiquez que la page actuelle est active.
        • Faites une pagination vers le prochain premier niveau d'AI.
        • Dans le fil d'Ariane, la page de premier niveau d'AI doit apparaître comme la page parent, précédée de la page d'index principale.
      • Contenu :
        • Utilisez la composante TTable des matières, si nécessaire.
        • Utilisez toute autre composante et fonctionnalité de conception standard, si nécessaire.
